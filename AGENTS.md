# AGENTS.md — Copilot CLI Toolkit

이 문서는 AI 에이전트(Copilot, Codex 등)가 이 저장소에서 작업할 때 반드시 따라야 하는 보안 하네스(Harness) 가이드라인을 정의합니다.

---

## 🔒 보안 하네스 (Security Harness)

### 1. 권한 없는 인프라 및 시스템 명령 금지

- `sudo`, `su`, `chmod 777`, `chown`, `mount`, `systemctl`, `service` 등 **시스템 관리자 권한이 필요한 명령을 실행하지 않습니다.**
- 컨테이너 탈출, 네트워크 설정 변경, 방화벽 규칙 수정 등 **인프라 수준의 변경을 시도하지 않습니다.**
- 환경 변수에 저장된 시크릿, 토큰, 자격 증명을 읽거나 출력하지 않습니다.
- `/etc`, `/var`, `/sys`, `/proc` 등 **시스템 디렉터리에 대한 쓰기를 시도하지 않습니다.**

### 2. Git 커밋 및 푸시 금지

- **`git commit`**, **`git push`** 등 **로컬 커밋과 원격 푸시를 직접 수행하지 않습니다.**
- 에이전트는 파일 변경만 수행하며, 커밋과 푸시는 반드시 **사용자가 직접** 실행합니다.
- `git commit`에 서명 키, 토큰, 자격 증명을 포함하지 않습니다.
- `.gitconfig`, `.git-credentials`, SSH 키 등 **Git 인증 관련 파일을 수정하거나 생성하지 않습니다.**
- `git remote set-url` 등으로 원격 저장소 URL을 변경하지 않습니다.

### 3. 알려지지 않은 외부 사이트 접근 금지

- 아래 **허용 목록(allowlist)에 명시된 도메인만** 접근할 수 있습니다.
- 허용 목록에 없는 외부 URL로의 `curl`, `wget`, `fetch`, HTTP 요청 등을 수행하지 않습니다.
- 사용자가 요청하더라도 알 수 없는 외부 서비스로 코드, 데이터, 시크릿을 전송하지 않습니다.

#### 허용된 외부 도메인

| 도메인 | 용도 |
|---|---|
| `github.com` | 저장소 참조, 이슈, PR |
| `api.github.com` | GitHub API 호출 |
| `registry.npmjs.org` | npm 패키지 레지스트리 |
| `pypi.org` | Python 패키지 레지스트리 |

> 새로운 외부 도메인 접근이 필요한 경우, 이 문서의 허용 목록을 먼저 업데이트한 후 사용하세요.

## 📋 일반 규칙

- 이 저장소의 루트 디렉터리(`/workspaces/copilot-cli-toolkit`) 범위 내에서만 파일을 생성·수정합니다.
- `.env`, `.secret`, `*credentials*` 등 민감한 파일을 커밋 대상에 포함하지 않습니다.
- 변경 사항은 항상 검증 가능한 상태로 유지하며, 기존 테스트가 있다면 통과를 확인합니다.
